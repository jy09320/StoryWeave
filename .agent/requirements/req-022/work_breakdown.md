# req-022 工作拆解

## 使用方式

本文件用于把 [`task.md`](./task.md) 和 [`implementation_plan.md`](./implementation_plan.md) 继续下钻为可直接执行的工作队列。

拆解层级：

1. 里程碑
2. 工作包
3. 具体任务
4. 完成定义

建议执行规则：

1. 同一时间只推进一个主里程碑。
2. 先完成基础设施，再做页面外观，再做智能交互。
3. 单次代码任务尽量落在一个明确工作包内，避免跨 3 个以上工作包同时改动。

## 里程碑 M1：工作台壳层

目标：建立双层侧边栏、上下文顶栏、右侧抽屉的统一 App Shell。

### 工作包 M1-W1：全局布局容器

1. 所有核心页面可挂载到统一壳层。
