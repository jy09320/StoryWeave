# req-022 用户路线优化计划

更新时间：2026-04-28

## 目标

这份计划不再从“页面列表”出发，而是从“作者如何连续使用 StoryWeave”出发，重新定义主链路、页面职责和后续优化优先级。

核心判断只有一句：

`StoryWeave` 的主目标不是“展示很多功能”，而是“让作者最快回到当前章节继续写，并在不中断心流的前提下完成查设定、调 AI、补资产”。

---

## 一、主链路定义

### 1. 继续写作

`Dashboard -> 最近项目/最近章节 -> Editor`

用户打开产品后的首要目标，不是浏览数据，而是恢复创作上下文。

### 2. 推进项目

`Dashboard / Project Workspace -> 章节列表 -> Editor`

当用户不是直接续写，而是要挑章节、调顺序、切状态时，先进入项目工作台，再进入章节。

### 3. 写作中查设定

`Editor -> 右侧抽屉 -> 返回正文`

这是高频动作，原则上不应跳出编辑器。

### 4. 写作中调用 AI

`Editor -> Bubble Menu / Slash Command / 右侧抽屉 -> 接受/放弃结果 -> 返回正文`

轻量 AI 属于正文的一部分，不应默认把用户送去独立工具页。

### 5. 写作中补资产

`Editor -> Characters / World -> 保存 -> 返回原章节`

只有当当前设定本身不完整时，才短暂离开编辑器维护真相源。

---

## 二、页面职责重定义

### Dashboard：继续写作启动器

唯一职责：

- 告诉用户上次写到哪
- 告诉用户现在最值得继续推进什么
- 把用户最快送回 Editor

不应该承担：

- 大量产品功能介绍
- 太重的项目管理
- 设定维护入口堆叠

### Project Workspace：项目总控台

唯一职责：

- 管项目结构
- 管章节列表与排序
- 管项目角色绑定
- 管世界观入口

不应该承担：

- 长时间正文创作
- 复杂 AI 处理
- 设定详情的高频维护

### Editor：正文生产面

唯一职责：

- 承担 80% 以上的真实创作过程
- 在这里完成写、改、查、调 AI、保存、切章

这是产品绝对中心。

### 右侧抽屉：参考与轻操作层

唯一职责：

- 给当前正文提供角色速查
- 给当前正文提供设定参考
- 从当前上下文发起轻量 AI 任务

不应该承担：

- 长表单维护
- 大段生成结果管理
- 替代角色库 / 世界观页

### AI Toolbox：重任务处理台

唯一职责：

- 长文本改写
- 多轮结果对照
- 历史复用
- 高级参数任务

边界：

- 轻 AI 在 Editor
- 重 AI 在 AI Toolbox

### Characters：角色真相源

唯一职责：

- 维护角色资料的最终可信版本

### World：设定真相源

唯一职责：

- 维护世界观资料的最终可信版本

### Settings：运行时与系统配置

唯一职责：

- 管模型、运行时、全局偏好

---

## 三、当前混乱点

### 1. 入口平权过度

现在 `Dashboard / Workspace / Editor / AI Toolbox / Characters / World` 看起来都像主页面，但真实主页面只有一个：`Editor`。

### 2. 页面职责重叠

同一类信息同时出现在：

- 项目工作台
- 右侧抽屉
- 角色库 / 世界观页
- AI 工具箱

用户会不知道“最终应该在哪里处理这件事”。

### 3. 跳页回路不够明确

从编辑器出发去 AI 工具箱、角色库、世界观后，用户容易丢失：

- 当前项目
- 当前章节
- 当前选区意图
- 返回后的落点

### 4. 编辑器外高频动作过多

如果查设定、轻改写、看角色这些动作都频繁跳页，创作心流就会断。

---

## 四、针对性优化策略

### 优先级 P0：明确唯一主链路

目标：

- 所有高频入口最终都指向 `Editor`
- 用户不再犹豫“我现在应该去哪”

动作：

1. Dashboard 首屏只保留一个强主动作：继续写作
2. Project Workspace 强化章节列表与管理，弱化正文预览
3. 所有“继续处理正文”的动作统一导向 Editor

### 优先级 P1：压缩编辑器外高频操作

目标：

- 在 Editor 内完成绝大多数日常动作

动作：

1. 角色速查、设定速查、轻量 AI 尽量留在右侧抽屉
2. Bubble Menu 与 Slash Command 覆盖正文内高频操作
3. 抽屉自动联动保持克制，只在真正需要时唤起

### 优先级 P1：切清 Editor 与 AI Toolbox 的边界

目标：

- 用户知道什么时候留在 Editor，什么时候去 AI Toolbox

规则：

1. Editor 负责选区级和当前章节级处理
2. AI Toolbox 负责长文本、多轮对照、历史复用和高级参数任务
3. 不再让两个地方同时承载同一类默认入口

### 优先级 P1：切清 Workspace 与 Editor 的边界

目标：

- 用户在项目工作台做管理，在编辑器做创作

规则：

1. Workspace 不承载重正文编辑
2. Workspace 的价值是“决定写哪一章”和“维护章节秩序”
3. 进入章节后，重心应立即转到 Editor

### 优先级 P2：建立统一跨页回路

目标：

- 从 Editor 跳出去，必须能回到原位置继续写

统一规则：

1. 所有从 Editor 出发的跳转都带上 `projectId`
2. 章节相关跳转都带上 `chapterId`
3. 选区相关跳转尽可能保留 `task`、`selectedText` 或等价上下文
4. 目标页提供明确的“返回当前章节”动作
5. 能回带结果的页面，优先回带到 Editor 而不是停留在目标页

---

## 五、可直接执行的优化批次

### Batch A：主页与项目入口收口

目标：

- 把“继续写作”变成首页唯一主叙事

任务：

1. 收紧 Dashboard 首屏信息密度
2. 把继续写作卡、最近章节、最近项目列表关系拉开
3. 把新建项目入口降级成辅助动作

### Batch B：Editor 中心化

目标：

- 减少跳页

任务：

1. 继续压缩右侧抽屉干扰
2. 补齐切章、查设定、轻 AI 的编辑器内闭环
3. 检查从 Editor 去 AI Toolbox / Characters / World 的返回链路

### Batch C：Project Workspace 去正文化

目标：

- 让 Workspace 更像总控台

任务：

1. 强化章节列表与状态管理
2. 强化角色绑定与世界观入口
3. 弱化正文摘要的大面积存在感

### Batch D：AI 边界收口

目标：

- 避免 AI 功能重叠

任务：

1. 明确编辑器内 AI 的默认入口
2. 明确 AI Toolbox 的重任务定位
3. 清理重复文案和重复动作

### Batch E：跨页状态回路统一

目标：

- 避免状态丢失

任务：

1. 统一编辑器跳出参数
2. 统一回带结果协议
3. 统一“返回当前章节”入口样式和位置

---

## 六、验收标准

当这套路线优化完成后，应该满足：

1. 用户打开首页后，3 秒内知道怎么继续写
2. 用户进入项目后，能迅速决定“写哪一章”或“补什么设定”
3. 用户写作时，大多数动作不需要离开 Editor
4. 用户离开 Editor 补资产或跑重 AI 任务后，能一跳回到原章节
5. 用户清楚每个页面的唯一职责，不再纠结“这件事到底该在哪做”

---

## 七、下一步建议

后续开发建议按下面顺序继续：

1. 先做 `Batch A + Batch B`
2. 再做 `Batch C + Batch D`
3. 最后做 `Batch E` 和真实浏览器验收

如果只选一件马上做，优先继续收 `Editor` 周边链路，因为这是当前主价值最密集的区域。
